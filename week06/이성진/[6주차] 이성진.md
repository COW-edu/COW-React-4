## css in CSS

### css in css란?

- 스타일을 정의하는 전통적인 방식으로 CSS 파일에 스타일을 정의하고 html 이나 컴포넌트에서 이를 받아와서 적용하는 방식이다.
- CSS 모듈을 사용하여 CSS를 모듈화 하는 방식이다.

> CSS 모듈은 CSS를 모듈화 하여 사용하는 방식이다. CSS 클래스를 만들면 자동으로 고유한 `클래스네임`을 만들어서 scope를 지역적으로 제한한다. 모듈화된 CSS를 번들러로 불러오면 다음과 같이 사용자가 정의했던 클래스 네임과 고유한 `클래스네임`으로 이루어진 객체가 반환된다.
> 

### css in CSS 특징

- **전역 스타일링** : CSS는 기본적으로 전역 스코프를 가지며, 다른 컴포넌트나 HTML 요소에 영향을 줄 수 있습니다. 또한 클래스를 지정해 스코프 를 제한해서 영향을 줄 수도 있다.

```jsx
/*style.css*/

.button {
background-color: gray;
color: white;
padding: 10px;
border: none;
border-radius: 5px;
}
```

```jsx
/* index.html*/

<button class="button">Click Me</button>
```

- **별도의 파일 관리**: 스타일은 `.css` 파일에 작성되고 HTML이나 자바스크립트 파일과는 별도로 관리된다.
- **브라우저 최적화**: CSS는 브라우저가 미리 컴파일하여 적용하기 때문에 CSS에 적합한 최적화가 이루어진다.

## css in JS

### css in JS란?

- **CSS in JS**는 자바스크립트 코드 내부에서 CSS를 정의하고 관리하는 방식이다.
- 리액트와 같은 자바스크립트 기반 프레임워크에서 주로 사용된다.

### css in JS 특징

- **충돌 걱정 감소:** css in JS를 사용하면 컴포넌트 별로 스타일을 따로 관리할 수 있기 때문에 다른 것과 충돌할 것을 방지할 수 있다.
- **유지보수 및 재사용 용이**: 컴포넌트와 스타일이 한 파일로 저장되어 있기 때문에 찾거나 수정하기 쉽다.
- **동적스타일링 가능:** 상황에 따라 스타일을 유동적으로 바꿀 수 있는데 , css in JS는 자바 스크립트 변수를 이용해서 스타일을 조건에 따라 변경할 수 있다.
****

## TailwindCSS

### Tailwind CSS란?

- Utility-First를 지향하는 **CSS 프레임워크** 이다.

> • Utility-First: **미리 세팅된 유틸리티 클래스를 활용하여 HTML 코드 내에서 스타일링**
> 
- CSS의 각 속성들을 클래스에 직관적으로 표현함으로써 효율적으로 사용할 수 있게 할 수  있다.

```css
// CSSi in CSS로 효현했을때
.button{
    background-color: green;
}
```

```jsx
//tailwind를 사용했을 때
<button class="bg-green-400" onClick={addItem}>
```

### tailwind의 장단점

1. 장점

- 코드의 직관성이 좋아 필요한 부분을 쉽게 찾아서 편집하기 좋다.
- 클래스 이름을 일일이 지정하지 않아도 되어서 css파일에서 찾아서 수정할 필요가 없다.
- 반복되는 스타일을 컴포넌트 추상화를 통해 재사용이 가능하다.

1. 단점

- 코드의 직관성이 좋은 반면 가시성이 떨어저 코드가 지저분 할 수 있다.
- 클래스가 출현한 빈도/ 순서에 상관없이 정의한 순서에 따라 스타일을 구성하기 때문에 CSS 속성들의 우선순위 문제가 있을 수 있다.
- 특정 프리픽스는 모든 CSS속성을 지원하지 않아서 tailwind.config에 직접 커스터 마이징 해야한다.
